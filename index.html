HTTP Back to section

HyperText Transfer Protocol

  • it is used to enable / establish communication between client and server
  • HTTP works as request-response protocol between server and client
  • a web browser is client

HTTP request methods Back to section

There are various methods of HTTP requests for example GET, POST, PUT, DELETE, UPDATE, CONNECT etc of which two methods are commonly used for request-response between client and server

  • GET
  • POST

Action attribute Back to section

  • this attribute is used to specify url of the server page to which we want to send over from data

GET method Back to section

  • In this method there is no security for data
  • This method can send only limited data (2MB)
  • It can carry raw data from client to server
  • GET -> Fastest response time

Checkbox Back to section

  • Checkboxes are used when more than one options are required to be selected
  • these are created using HTML input tag type="checkbox"

GET method Back to section

Note:

  • GET requests can be cached
  • GET remain in browser history
  • GET requests can be bookmarked
  • GET requests should not deal with sensitive data
  • GET requests length is restricted
  • GET requests have excellent performance
  • GET requests can not upload images

POST method Back to section

  • POST requests can not be cached
  • POST requests can not be bookmarked
  • POST requests can deal with sensitive data
  • POST requests have no limit for length of data
  • POST requests have good performance
  • POST requests can upload images
  • POST requests do not remain in browser history

HTML status messages (codes) Back to section

  • 1xx -> 100 to 199 : Informational
  • 2xx -> 200 to 299 : Successful
  • 3xx -> 300 to 399 : Redirection
  • 4xx -> 400 to 499 : Client side server
  • 5xx -> 500 to 599 : Server side error

CSS Back to section

  • CSS stands for Cascading Stylesheet.
  • It is a stylesheet language used for describing the presentation of document written in a markup language like HTML.
  • CSS is a cornerstone technology for world wide web alongside HTML and JavaScript.
  • CSS is designed to enable separation of presentation and contents.
  • All styles are stored in CSS files.
  • CSS files must have extension as '.css' only.

Why CSS? Back to section

  • it allows much richer presentation appearance.
  • it reduces workload by centralizing commands for visual appearance instead of scattered ones.
  • Same styles can be reused for multiple documents.

History Back to section

  • First version of CSS was released in 1996.
  • Second version of CSS was released in 1998, with improved table features.
  • third version of CSS was released in 2008 and currently in use with improved support for modules, it also supports modern browsers.
  • fourth version was released in 24th march 2017, which is faster than CSS3

Types of CSS Back to section

  1. Inline CSS -> Highest priority
    • can be applied to only that tag
  2. Internal CSS -> Medium priority
    • can be applied to only that document
  3. External CSS -> Lowest priority
    • can be applied globally

Note: Download and install Visual Studio Code

CSS syntax (rule-set) Back to section

  • CSS syntax is selector and declaration block
  • The selector points to the HTML element you want to style
  • Each declaration includes CSS property and value pair
  • Properties and values are separated by colon(:)
  • Each property and value separated by semicolon(;)
  • For Example

    • h1
      {
      color: red;
      font-size: 12px;
      }
      where

      h1 is a selector

      color is property

      red is value

      font-size is property

      12px is value
