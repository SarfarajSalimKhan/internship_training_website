Tag

Text placed between '<' and '>' is called as tag

Example: <html>

Types of tag

  • Pair (<> </>)

    • The tags that have both opening and closing tags are called as paired tags.

    Example:

    • <html>
          ...
      </html>
    • <body>
          ...
      </body>
  • Single (</> / </>)

Note: closing tags start with '/'

Non-paired tags

  • The tags that have only opening tag but no closing tag are called as non-paired tags
  • Also called as self closed tags or forcefully tags or empty tags or singular tags

    • Example:

    • Break tag (line break)
      • <br> or <br/>
    • Horizontal rule (horizontal line)
      • <hr> or <hr/>
    • Image tag
      • <img> or <img/>

HTTP

HyperText Transfer Protocol

  • it is used to enable / establish communication between client and server
  • HTTP works as request-response protocol between server and client
  • a web browser is client

HTTP request methods

There are various methods of HTTP requests for example GET, POST, PUT, DELETE, UPDATE, CONNECT etc of which two methods are commonly used for request-response between client and server

  • GET
  • POST

Action attribute

  • this attribute is used to specify url of the server page to which we want to send over from data

GET method

  • In this method there is no security for data
  • This method can send only limited data (2MB)
  • It can carry raw data from client to server
  • GET -> Fastest response time

Checkbox

  • Checkboxes are used when more than one options are required to be selected
  • these are created using HTML input tag type="checkbox"

GET method

Note:

  • GET requests can be cached
  • GET remain in browser history
  • GET requests can be bookmarked
  • GET requests should not deal with sensitive data
  • GET requests length is restricted
  • GET requests have excellent performance
  • GET requests can not upload images

POST method

  • POST requests can not be cached
  • POST requests can not be bookmarked
  • POST requests can deal with sensitive data
  • POST requests have no limit for length of data
  • POST requests have good performance
  • POST requests can upload images
  • POST requests do not remain in browser history

HTML status messages (codes)

  • 1xx -> 100 to 199 : Informational
  • 2xx -> 200 to 299 : Successful
  • 3xx -> 300 to 399 : Redirection
  • 4xx -> 400 to 499 : Client side server
  • 5xx -> 500 to 599 : Server side error

CSS

Cascading Style Sheet

  • It is a stylesheet language used for describing the presentation of document written in a markup language like HTML
  • CSS is a cornerstone technology for world wide web alongside HTML and JavaScript
  • CSS is designed to enable separation of presentation and contents
  • All styles are stored in CSS files
  • CSS files must have extension as ".css" only

Why CSS?

  • it allows much richer presentation appearance
  • it reduces workload by centralizing commands for visual appearance instead of scattered ones
  • some styles can be reused for multiple documents

History

  • First version of CSS was released in 1996
  • Second version of CSS was released in 1998 with improved table features
  • third version of CSS was released in 2008 and currently in use with improved support for modules, it also supports modern browsers
  • fourth version was released on 24th march 2017 which is faster than CSS3

Types of CSS

  1. Inline CSS -> Highest priority
    • can be applied to only that tag
  2. Internal CSS -> Medium priority
    • can be applied to only that document
  3. External CSS -> Lowest priority
    • can be applied globally

Note: Download and install Visual Studio Code

CSS syntax (rule-set)

  • CSS syntax is selector and declaration block
  • The selector points to the HTML element you want to style
  • Each declaration includes CSS property and value pair
  • Properties and values are separated by colon(:)
  • Each property and value separated by semicolon(;)
  • For Example

    • h1
      {
      color: red;
      font-size: 12px;
      }
      where

      h1 is a selector

      color is property

      red is value

      font-size is property

      12px is value

CSS selectors

  • CSS selectors are used to select the contents you want to style
  • Types
    • Element selector
    • Id selector
    • Class selector
    • Universal selector
    • Group selector

Element selector

the element selector selects the HTML element by name

Syntax

element
{
style1;
style2;
...
}

p
{
text-align: center;
color: blue;
}

Id selector

  • it uses id attribute of HTML element
  • to select a selector with specific id (#) is used
  • Syntax
    • #id
      {
      style1;
      style2;
      ...
      }
    • #warning
      {
      color: orange;
      }

Class selector

  • the class selector selects HTML elements with specific class attribute
  • it is used with a "period" symbol (.dot) followed by classname
  • Syntax
    • .class_name
      {
      style1;
      style2;
      ...
      }
    • Example:
      .cool
      {
      text-align: right;
      color: #002366;
      }

Class selector for specific element

if you want to specify only one specific element should be affected then you should use element name with class name

h1.cool
{
...
}

Universal selector

  • The universal selector is used as wildcard selector
  • It selects all elements in page
  • It is denoted by "*" symbol
  • *
    {
    style1;
    style2;
    ...
    }

Group selector

  • Group selector is used to select all elements with same style definition commas are used to separate selectors in grouping
  • Example:
    h2
    {
    text-align: center;
    color: maroon;
    }
  • h3
    {
    text-align: center;
    color: maroon
    }
  • h1, h2
    {
    text-align: center;
    color: maroon
    }

    Add user credentials

    • add name -> $git config --global user.name ""
    • add email -> $git config --global user.email ""

    Create files to local repository

    • $touch <>
    • $touch sample.txt
    • sample.txt is unsaved / untracked / uncommited

    • $git add .
    • $git commit

    • press "Esc"
    • type ":wq"

    • $git commit -m <message>
    • check log -> $git log

    Create branches

    • $git branch "demo" -> by default all files from master are available here
    • $git checkout demo

    Merge branches

    • $git checkout master
    • $git merge demo
